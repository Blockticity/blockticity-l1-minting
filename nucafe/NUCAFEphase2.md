## Post-Mint SVG Correction

Two issues were identified after minting and corrected via S3 SVG re-upload (no re-mint required):

1. **"Lot No. / ICO No." field removed** — This field was present in the initial SVG rendering but was not part of the approved bag COA field set. Removed from all 350 SVGs.
2. **Shipment Ref corrected** — Changed from "SC 2526.040" to "SC 2526.004" across all 350 SVGs.

**Why no re-mint was needed:** The on-chain tokenURI contains a JSON metadata object where the `image` field is an S3 URL pointing to the SVG — the SVG is not embedded as base64 in the tokenURI. Updating the S3-hosted SVG corrected the visual certificate for all viewers (app.blockticity.ai, QR scans, direct links). The on-chain `blockticity.fields` JSON still contains the original field values, but the content hash verification integrity is preserved — the hash validates against the on-chain data as minted.

All 350 SVGs were re-rendered, re-uploaded to S3 with `Cache-Control: no-cache` headers, and COA thumbnails re-extracted.

## Farm Attestation

### Computation Method
The Farm Attestation Hash cryptographically links all 350 bag certificates to the 42 source farm certificates from Phase 1:

1. Fetch all 42 farm content hashes from on-chain `tokenURI()` data (tokens #762114–#762155)
2. Sort the 42 content hashes lexicographically (ascending)
3. Strip `0x` prefixes and concatenate as raw hex string
4. Compute SHA-256 hash of the concatenated string
5. Prefix with `0x` → Farm Attestation Hash

**Result:** `0x27950770fb562dbde6ff3017d5819b374e7a30cc31ccc1c9e37476efff840892`

## Methodology

### Automated Pipeline
Phase 2 used a fully automated Node.js pipeline (`phase2-pipeline/`) consisting of 6 sequential scripts:

1. **01-compute-attestation.js** — Fetched 42 farm content hashes from on-chain tokenURI data, computed Farm Attestation Hash
2. **02-generate-all.js** — Generated 350 publicJSON documents, content hashes, GVE codes, QR codes, and SVG certificates. Passed 10/10 determinism check.
3. **03-test-mint.js** — Test minted BAG-001, verified on-chain tokenURI matches local metadata
4. **04-batch-mint.js** — Batch minted remaining 349 bags in batches of 10 with checkpoint-based resume. Individual `mintURI()` calls (`batchMintURI` fails with large tokenURIs). Per-mint checkpoint saves with 60s timeout and 3-retry logic.
5. **05-qa-gallery.js** — QA HTML gallery with Phase 1 cross-reference
6. **06-verify-all.js** — On-chain verification of all 350 tokens: contentHash, attestation hash, and farmTokenIds confirmed

### On-Chain Verification
- All 350 token URIs fetched and decoded from chain
- Content hashes recomputed and matched
- Farm Attestation Hash recomputed from 42 source hashes and matched
- Source farm token IDs confirmed in every bag's `sourceAttestation` object
- **Result: 350/350 verified, 0 failures**
